## Assessment load + editor initialization

Trigger: Open editor for an assessment with 0 questions
Correct Behavior: Clear empty state (“No questions yet”) + prominent “Add Question”
False Behavior: Blank section, broken layout, or crash

Trigger: Assessment fetch is slow
Correct Behavior: Loading state; disable save/add/delete until data loaded
False Behavior: Actions enabled with undefined state; partial render; infinite spinner

Trigger: Assessment fetch fails (404/500/timeout)
Correct Behavior: Error banner + retry; 404 routes back to list; no data loss
False Behavior: White screen, endless loading, or editing stale cached data

Trigger: Backend returns partial/malformed assessment schema
Correct Behavior: Safe defaults + validation warnings; editor remains usable
False Behavior: Runtime exception or silently dropped fields/questions

Trigger: Very large assessment (100+ questions, long options)
Correct Behavior: Responsive UI; virtualization/pagination if needed; save remains reliable
False Behavior: UI freezes, typing lag, or browser memory blow-up

## Key + metadata (title/description)

Trigger: Key is edited to a value that already exists
Correct Behavior: Validation blocks; server returns 409; clear “Key already in use”
False Behavior: Duplicate keys created or edits saved but break lookup/routing

Trigger: Key contains invalid characters/spaces/reserved words
Correct Behavior: Client + server enforce pattern; actionable error message
False Behavior: Key accepted but later breaks publishing/submissions/URLs

Trigger: Title/description includes Unicode (Greek), emojis, or long text
Correct Behavior: Renders correctly; truncation only where intended; round-trip safe
False Behavior: Garbled text or content changes after save/reload

Trigger: Description uses WYSIWYG content (HTML)
Correct Behavior: Sanitized on save; renders identically on learner side
False Behavior: XSS, broken markup, or formatting drift on reload

## Question CRUD (add/remove/reorder)

Trigger: Add Question repeatedly / double-click
Correct Behavior: Creates one question per intent; button disabled in-flight (if server-backed)
False Behavior: Duplicate phantom questions or inconsistent numbering

Trigger: Delete a question with existing submissions/responses
Correct Behavior: Enforce policy (block, soft-delete, or versioning); explain impact
False Behavior: Hard delete corrupts historical submissions or breaks analytics

Trigger: Reorder questions (drag/drop or arrows)
Correct Behavior: Order persists after save/reload; stable numbering; no lost data
False Behavior: UI order differs from persisted order; reordering duplicates/drops questions

Trigger: Collapse/expand questions (accordion UI)
Correct Behavior: State is predictable; does not reset user input; keyboard accessible
False Behavior: Collapsing loses edits or expands wrong question

Trigger: Edit a question while another save is in-flight
Correct Behavior: Prevent conflicting operations or queue; ensure final saved state matches UI
False Behavior: Last edits lost due to race conditions

## Question identity (IDs) and referencing

Trigger: Two questions share the same Question ID
Correct Behavior: Block save; highlight collisions; explain uniqueness requirement
False Behavior: Save succeeds but submissions map answers to wrong question

Trigger: Question ID changed after submissions exist
Correct Behavior: Disallow or require versioning/migration; explicit warning
False Behavior: Existing submissions become unmapped or analytics break

Trigger: Autogenerated IDs (q1, q2…) after deletion/reorder
Correct Behavior: IDs remain stable; do not reassign existing IDs automatically
False Behavior: IDs renumber causing answer mapping corruption

## Question types + type switching

Trigger: Switch type (Choice -> Text -> Scale etc.)
Correct Behavior: Convert safely or require confirmation; preserve compatible fields where possible
False Behavior: Silent destructive loss of options/validation rules

Trigger: Type requires options but options are empty
Correct Behavior: Validation blocks publish/save (as per rules); explicit “Options required”
False Behavior: Question saved/published but cannot be answered

Trigger: Type supports multiple selections vs single selection
Correct Behavior: UI enforces correct control; payload stores correct mode
False Behavior: Single-select rendered but stored as multi (or vice versa)

Trigger: Type includes numeric constraints (min/max/step)
Correct Behavior: Validate constraints (min<=max, step>0); correct coercion rules
False Behavior: Invalid constraints saved; runtime errors in learner UI

Trigger: Type requires correct answer / scoring (if you support scoring)
Correct Behavior: Enforce only for scorable types; validate referenced option exists
False Behavior: Score references missing option or wrong type; scoring fails silently

## Required/optional logic

Trigger: Toggle Required on/off
Correct Behavior: Persists after save; learner validation matches admin setting
False Behavior: Required flag ignored on learner side or flips back on reload

Trigger: Required question has empty label/text
Correct Behavior: Block publish; highlight missing label
False Behavior: Published assessment contains required blank question causing dead-end

## Options editor (for choice/multi-choice)

Trigger: Options contain empty lines / whitespace-only entries
Correct Behavior: Trim and ignore empties or block with clear error (consistent rule)
False Behavior: Empty options appear to learners as blank selectable items

Trigger: Duplicate options (“Yes” twice)
Correct Behavior: Either allowed with stable distinct IDs, or blocked with warning
False Behavior: Answers ambiguous; scoring/analytics mis-attribute

Trigger: Options with leading/trailing spaces
Correct Behavior: Normalized consistently on save and display
False Behavior: “Option” and “Option ” treated as different unexpectedly

Trigger: Very large option sets (100+)
Correct Behavior: UI stays responsive; scrolling usable; bulk paste supported
False Behavior: Lag, input drop, or save payload truncation

Trigger: Paste options from external source (newline-separated)
Correct Behavior: Splits reliably; sanitizes; preserves intended order
False Behavior: Merges lines, introduces weird characters, or drops items

## Validation + publishing workflow

Trigger: Publish assessment with invalid schema (missing question text, invalid IDs, no options)
Correct Behavior: Publish blocked; error summary points to exact question/field
False Behavior: Published but unusable assessment or learner runtime errors

Trigger: Publish clicked twice / rapid publish/unpublish toggles
Correct Behavior: Idempotent endpoint; UI disables in-flight; final state consistent
False Behavior: Conflicting states, duplicate version creation, or stale UI

Trigger: Editing a published assessment (policy dependent)
Correct Behavior: Either locked, or creates a new version/draft; clear labeling
False Behavior: Live edits mutate active assessment and invalidate existing submissions

Trigger: Server returns canonical normalized payload after save (e.g., trims text)
Correct Behavior: UI updates to canonical state; reload matches exactly
False Behavior: “Drift” where each save subtly changes content unexpectedly

## Conditional logic / branching (if supported)

Trigger: Add display logic (show Q2 only if Q1=...)
Correct Behavior: Only valid dependencies allowed (no forward refs if disallowed); cycles blocked
False Behavior: Circular dependencies or references to deleted questions

Trigger: Delete a question that other logic depends on
Correct Behavior: Block delete or automatically remove/update dependent rules with confirmation
False Behavior: Dangling rules break runtime rendering

---

## Concurrency + drafts

Trigger: Two admins edit same assessment simultaneously
Correct Behavior: Conflict detection (ETag/version) or clear “updated elsewhere” prompt
False Behavior: Silent overwrite of changes

Trigger: User navigates away with unsaved changes
Correct Behavior: Dirty-state confirm; optional autosave draft
False Behavior: Silent data loss

Trigger: Save succeeds but network drops before response
Correct Behavior: On reconnect/refresh, editor reconciles with server state safely
False Behavior: User unsure if saved; duplicates created by retry

---

## Submissions + schema integrity

Trigger: Existing submissions when editor changes structure (delete/reorder/change type)
Correct Behavior: Enforce safe policies (versioning, lock, migration); keep historical data queryable
False Behavior: Analytics/submissions pages crash or show misaligned answers

Trigger: “Required” changed after submissions exist
Correct Behavior: Only affects new submissions; old submissions remain valid
False Behavior: Old submissions flagged invalid or blocked from viewing

---

## Security + sanitization

Trigger: Question text/options include HTML/script payload
Correct Behavior: Escaped/sanitized on save and render; no XSS
False Behavior: Stored XSS on learner page or admin submissions view

Trigger: Error messages reflect user-provided content
Correct Behavior: Safely escaped; does not render markup
False Behavior: XSS via validation messages/toasts

---

## Accessibility + UX

Trigger: Keyboard-only editing (tab through fields, add/delete question, reorder)
Correct Behavior: Fully operable; focus management correct; visible focus
False Behavior: Focus lost, trapped, or actions unreachable

Trigger: Screen reader usage
Correct Behavior: Proper labels, ARIA for accordions/buttons; error summary navigable
False Behavior: Unlabeled controls; errors not announced

Trigger: Small viewport / responsive layout
Correct Behavior: Fields stack; question cards scroll; action buttons remain reachable
False Behavior: Actions overlap, off-screen buttons, or horizontal scroll required for basics
